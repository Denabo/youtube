# 🎬 Multi-Platform Short Video Pipeline

Проект для **потоковой (пакетной) сборки коротких видео** под:
- YouTube Shorts
- TikTok
- Instagram Reels
- VK Клипы

Поддерживается:
- наложение фона;
- наложение баннера (с хромакеем);
- фоновая музыка;
- автосубтитры через Whisper;
- пакетная очередь обработки;
- отдельная утилита нарезки по тайм-кодам.

---

## 📁 Актуальная структура

```text
youtube/
├── main.py                # Точка входа (терминальный UI)
├── terminal_ui.py         # Меню очереди и настроек обработки
├── batch_processor.py     # Пакетная обработка + рендер под платформы
├── subtitles.py           # Генерация и отрисовка субтитров
├── chroma_key.py          # Удаление розового/маджента фона у баннера
├── video_cutter.py        # Нарезка видео по списку тайм-кодов
├── config.py              # Вся конфигурация пайплайна
├── main_scripts/
│   └── Obrezka.py         # Совместимость, прокси к video_cutter.py
├── input/
│   ├── clips/             # Исходные клипы
│   ├── music/             # Музыка
│   ├── backgrounds/       # Фоны
│   └── banners/           # Баннеры
├── output/
│   ├── youtube/
│   ├── tiktok/
│   ├── instagram/
│   └── vk/
└── processed/             # Уже обработанные клипы
```

## 🚀 Быстрый старт

1. Положите материалы в папки `input/*`.
2. Запустите интерфейс:

```bash
python main.py
```

3. В меню:
- создайте очередь;
- выберите режим сборки;
- выберите платформы (`все` или конкретные);
- запустите обработку.

Результаты попадут в `output/<platform>/`.

---

## 🧠 Режимы обработки

Настраиваются в `config.py` (`PROCESSING_MODES`):
- обрезка под 9:16;
- фон + уменьшенный клип;
- баннер поверх;
- универсальный режим (комбинируется флагами).

## 🩷 Настройка хромакея под розовый фон

По умолчанию проект настроен на удаление розового/маджента фона:

```python
CHROMA_COLOR = (255, 0, 255)
```

Параметры в `config.py`, которые обычно подбирают:
- `CHROMA_COLOR` — целевой цвет для удаления;
- `CHROMA_TOLERANCE` — насколько агрессивно удалять близкие оттенки;
- `CHROMA_EDGE_BLUR` — размытие края маски, чтобы убрать «лесенку».
- `BANNER_TOP_MARGIN` — отступ баннера от верхнего края (в пикселях).
- `BANNER_VERTICAL_OFFSET` — дополнительный сдвиг баннера по вертикали (можно отрицательный / можно увести за границы экрана).
- `WHISPER_LANGUAGE = None` — автоопределение языка речи для субтитров.

Если в исходнике фон не чисто розовый (например, фуксия/сиреневый), увеличьте `CHROMA_TOLERANCE` на 10–30 пунктов.

Чтобы разместить баннер почти у самого верхнего края, обычно ставят:
- `BANNER_TOP_MARGIN = 0`
- `BANNER_VERTICAL_OFFSET = 0`
- размер баннера оставляется исходным (без автоскейла), после обрезки прозрачных полей.

Примеры:
- `BANNER_VERTICAL_OFFSET = -120` — баннер частично уйдёт выше верхней границы.
- `BANNER_VERTICAL_OFFSET = 300` — баннер опустится ниже.

## 🔬 Как работает код поэтапно (весь пайплайн)

### 1) Запуск приложения
- Вы запускаете `python main.py`.
- `main.py` печатает инструкцию и передаёт управление в `terminal_ui.main()`.

### 2) Инициализация интерфейса и сканирование файлов
- `TerminalUI` создаёт `BatchProcessor`.
- `BatchProcessor` сканирует:
  - `input/clips/*` — исходные ролики;
  - `input/music/*` — треки.
- Далее определяет следующий номер `clip_<N>_...` по уже готовым файлам в `output/`.

### 3) Сбор очереди
- В терминальном меню можно:
  - выбрать режим обработки;
  - выбрать музыку (одну, без музыки или «случайно для каждого»);
  - выбрать платформы (YouTube/TikTok/Instagram/VK).
- На каждый клип формируется запись в очереди:
  - индекс клипа;
  - индекс музыки;
  - режим;
  - список платформ.

### 4) Старт пакетной обработки
- `BatchProcessor.process_queue()` проходит по очереди.
- Для каждого элемента создаётся `VideoProcessor`.
- Дополнительно пытается построить субтитры через `generate_subtitles()`.

### 5) Обработка одного клипа
- `VideoProcessor.process()`:
  1. Берёт профиль платформы (разрешение/FPS);
  2. Открывает исходный клип;
  3. Собирает видеослои через `_mode_universal()`;
  4. Добавляет музыку через `_add_music()` (если выбрана);
  5. Накладывает субтитры (если распознавание прошло успешно);
  6. Рендерит в `output/<platform>/clip_<N>_<platform>.mp4`.

### 6) Логика сборки кадра (`_mode_universal`)
В зависимости от флагов выбранного режима:
- **background=True**  
  Загружает первый рабочий фон из `input/backgrounds/*.mp4`, подгоняет по размеру, при необходимости лупит.
- **crop=True**  
  Обрезает исходный клип под вертикальный формат 9:16.
- **resize_clip=True**  
  Уменьшает клип, позиционирует выше центра.
- **banner=True**  
  Берёт баннер из `input/banners/*.mp4`, убирает розовый фон через `chroma_key()`, масштабирует и ставит сверху.

Итоговые слои объединяются в `CompositeVideoClip`.

### 7) Как работает удаление розового фона (`chroma_key.py`)
- Для каждого кадра считается расстояние каждого пикселя до `CHROMA_COLOR`.
- Формируется маска прозрачности:
  - пиксели, похожие на розовый, делаются прозрачными;
  - «не-розовые» остаются непрозрачными.
- Дополнительно включена эвристика «маджента-доминантности»:
  - высокий `R` и `B`,
  - низкий `G`.
- Затем:
  - сглаживание края (`gaussian_filter`);
  - пороговая чистка от полупрозрачных артефактов.
- Маска прикладывается к баннеру как альфа-канал.

### 8) Работа с аудио
- Оригинальный голос клипа сохраняется (`VOICE_VOLUME`).
- Музыка:
  - обрезается или зацикливается под длину ролика;
  - микшируется с голосом через `CompositeAudioClip`.

### 9) Субтитры (если включены)
- Из видео извлекается WAV 16kHz.
- Whisper распознаёт речь и возвращает сегменты с временем.
- Текст дробится на короткие фразы (`SUBTITLE_WORDS_PER_PHRASE`).
- Для каждой фразы рисуется прозрачный PNG-слой с подложкой и текстом.
- Слои накладываются в нужные интервалы времени.

### 10) Завершение цикла
- Готовый файл сохраняется в `output/...`.
- Исходный клип переносится в `processed/`, чтобы повторно не обрабатывался.
- После всей очереди печатается финальный отчёт.

## ✂️ Нарезка по списку тайм-кодов

### Вариант A — список в файле

Создайте `cuts.txt` (первая строка `start,end,name` — опционально):

```text
start,end,name
00:00:05,00:00:15,intro
00:00:20,00:00:35,hook
00:01:00,00:01:20,
```

Запуск:

```bash
python video_cutter.py --input input/clips/source.mp4 --output output/cuts --ranges-file cuts.txt
```

### Вариант B — список в коде

Отредактируйте `CUT_RANGES` в `video_cutter.py` и запустите без `--ranges-file`.
